SWIR(TBD)

The SWIR Model [1] extends the classic SIR framework by introducing a Weakened (W) state between Susceptible (S) and Infected (I). Infection spreads only through links between neighboring nodes in a graph \(G=(V,E)\). Each node is in one of four states: \(S\) (susceptible), \(W\) (weakened), \(I\) (infected), or \(R\) (recovered). At a high level, a susceptible node can either be weakened or directly infected by its infected neighbors; weakened nodes can subsequently become infected; infected nodes recover and become immune (R). The model is parameterized by three per-contact rates: \(\kappa\) (direct infection), \(\mu\) (weakening), and \(\nu\) (weak-to-infected).

Implementation

Node transitions follow rules:

  1. if a \(S\) state node has \(I\) state neighbors, i) each \(I\) state neighbor exposes the \(S\) state node to W with probability \(\mu\) (S→W) or ii) each \(I\) state neighbor exposes the \(S\) state node to I with probability \(\kappa\) (S→I);

  2. the \(W\) state node has \(I\) state neighbors, each \(I\) state neighbor exposes the \(W\) state node with probability \(\nu\) (W→I);

  3. the \(I\) state node is recovered to the \(R\) state with probability 1 (I→R).

We represent node states with three Boolean indicator vectors \(h, w, r \in \{0,1\}^N\). Let \(N(i)\) be the neighbor set of node \(i\). The update at discrete step \(k\) is performed in the following stages (matching the reference implementation):

  • \(h_i=1\) indicates node \(i\) is infected or weakened; \(h_i=0\) indicates susceptible or recovered.

  • \((h_i,w_i,r_i)=(1,0,0)\) denotes infected, \((h_i,w_i,r_i)=(1,1,0)\) denotes weakened.

  • \((h_i,w_i,r_i)=(0,0,1)\) denotes recovered, \((h_i,w_i,r_i)=(0,0,0)\) denotes susceptible.

    1. Each infected neighbor \(j\) of node \(i\) transmits a log-probability contribution

For each edge \(j\to i\), an infected neighbor contributes to the log-survival; the per-step infection probability for node \(i\) is

\[m_{ji}^{(k)} = \mathbf{1}\!\left(h_j^{(k-1)}=1 \land w_j^{(k-1)}=0\right) \cdot \log(1-\kappa), \qquad I_i^{(k)} = 1 - \exp\!\left( \sum_{j\in N(i)} m_{ji}^{(k)} \right),\]

applied only to currently susceptible and not recovered nodes.

  1. Weakening (S→W)

Analogously, the probability to become weakened is

\[\widetilde{m}_{ji}^{(k)} = \mathbf{1}\!\left(h_j^{(k-1)}=1 \land w_j^{(k-1)}=0\right) \cdot \log(1-\mu), \qquad W_i^{(k)} = 1 - \exp\!\left( \sum_{j\in N(i)} \widetilde{m}_{ji}^{(k)} \right),\]

again only for nodes that are susceptible and not recovered.

  1. Weakened to infected (W→I)

A weakened node may become infected via infected neighbors with rate \(\nu\):

\[\widehat{m}_{ji}^{(k)} = \mathbf{1}\!\left(h_j^{(k-1)}=1 \land w_j^{(k-1)}=0\right) \cdot \log(1-\nu), \qquad WI_i^{(k)} = 1 - \exp\!\left( \sum_{j\in N(i)} \widehat{m}_{ji}^{(k)} \right),\]

applied only to currently weakened nodes (\(w_i^{(k-1)}=1\)).

  1. Recovery (I→R)

In this implementation, infected and not-weakened nodes deterministically recover at each step (no parameter \(\gamma\)). We model this as

\[r_i^{(k)} = r_i^{(k-1)} \;\lor\; \big( h_i^{(k-1)} = 1 \land w_i^{(k-1)} = 0 \big),\]

and recovered nodes remain immune: they neither get weakened nor re-infected.

  1. Sampling and state update

Let \(U_i^{\mathrm{inf}}, U_i^{\mathrm{wk}}, U_i^{\mathrm{wi}} \sim \mathrm{Uniform}(0,1)\) be independent. The indicators are updated as

\[\begin{split}\begin{aligned} w_i^{(k)} &= \begin{cases} 1, & \text{if } w_i^{(k-1)}=0 \land r_i^{(k-1)}=0 \land U_i^{\mathrm{wk}} < W_i^{(k)}, \\ 0, & \text{if } w_i^{(k-1)}=1 \land U_i^{\mathrm{wi}} < WI_i^{(k)}, \\ w_i^{(k-1)}, & \text{otherwise}, \end{cases} \\ h_i^{(k)} &= \begin{cases} 1, & \text{if } h_i^{(k-1)}=0 \land r_i^{(k-1)}=0 \land U_i^{\mathrm{inf}} < I_i^{(k)}, \\ 1, & \text{if } w_i^{(k-1)}=1 \land U_i^{\mathrm{wi}} < WI_i^{(k)}, \\ 0, & \text{if } h_i^{(k-1)}=1 \land w_i^{(k-1)}=0 \text{ (recovery)}, \\ h_i^{(k-1)}, & \text{otherwise}, \end{cases} \end{aligned}\end{split}\]

which mirrors the code path: S→W, S→I, W→I, I→R.
